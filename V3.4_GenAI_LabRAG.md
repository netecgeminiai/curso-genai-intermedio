### [01:10-02:40] PASO 1: CARGA Y CHUNKING INTELIGENTE
**[VISUAL: El instructor escribe código Python, importando módulos y cargando el PDF]**

Primero, importamos los cargadores y el separador de texto de LangChain.

```python
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 1. Cargamos el PDF completo a la memoria
print("Cargando documento...")
loader = PyPDFLoader("politicas_empresa.pdf")
documentos_completos = loader.load()
```

Ahora, aplicamos la teoría que vimos en el video anterior: **El Chunking**. No podemos meter todo el PDF en un solo vector. Vamos a cortarlo con el `RecursiveCharacterTextSplitter`, que es inteligente y no rompe oraciones a la mitad.

```python
print("Aplicando Chunking...")
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,     # Cortar pedazos de 500 caracteres
    chunk_overlap=50    # Repetir los últimos 50 caracteres para no perder el hilo
)

# Convertimos las páginas gigantes en pequeños fragmentos
chunks = text_splitter.split_documents(documentos_completos)
print(f"Documento dividido en {len(chunks)} chunks.")
```

---

### [02:40-04:00] PASO 2: EMBEDDINGS Y BASE DE DATOS VECTORIAL
**[VISUAL: Se importa Chroma y OpenAIEmbeddings. Se muestra cómo los textos se "hunden" en la DB]**

Tenemos nuestros fragmentos de texto en Python. Ahora necesitamos traducirlos a matemáticas (Embeddings) y guardarlos en una Base de Datos Vectorial.

Usaremos el modelo de Embeddings de OpenAI y Chroma, que es una base de datos que corre directamente en memoria, sin necesidad de instalar servidores complejos en este laboratorio.

```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
import os

# Asegúrate de tener exportada tu variable de entorno OPENAI_API_KEY
embeddings_model = OpenAIEmbeddings(model="text-embedding-3-small")

print("Generando vectores y guardando en ChromaDB...")
vectorstore = Chroma.from_documents(
    documents=chunks, 
    embedding=embeddings_model, 
    persist_directory="./chroma_db_local" # Guarda la BD en el disco duro
)
```

¡Boom! Detrás de escena, este script envió todos nuestros textos a la API de OpenAI, recibió los miles de números decimales de vuelta, y Chroma los estructuró en su bóveda tridimensional en nuestro disco duro. 

Ya no tenemos que volver a hacer este paso. La "R" (Recuperación) de nuestro RAG está lista para la acción.

### [04:00-05:20] PASO 3: EL RECUPERADOR (RETRIEVER) Y EL PROMPT
**[VISUAL: Se crea la función que recibe la pregunta y extrae el texto de la DB]**

Ahora, vamos a simular que el usuario hace una pregunta.

```python
pregunta_usuario = "¿Cuántos días de vacaciones me tocan si llevo 3 años trabajando?"
```

Primero, necesitamos configurar el buscador (Retriever). Le diremos a Chroma que busque en sus vectores, y nos devuelva los **3 fragmentos** que matemáticamente más se parezcan a la pregunta del usuario.

```python
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
```

Y aquí viene el corazón de RAG: **El System Prompt**. Le damos instrucciones estrictas al LLM para que no alucine.

```python
from langchain_core.prompts import ChatPromptTemplate

template = """
Eres un asistente de Recursos Humanos de la empresa.
Usa SOLO el siguiente contexto para responder a la pregunta del empleado.
Si la respuesta no está en el contexto, di "No tengo información sobre eso". No inventes datos.

Contexto:
{context}

Pregunta del empleado: {question}
"""
prompt = ChatPromptTemplate.from_template(template)
```

---

### [05:20-06:30] PASO 4: ENSAMBLANDO LA CADENA Y GENERANDO
**[VISUAL: Importando el LLM, armando la cadena (Chain) en LangChain usando LCEL (LangChain Expression Language)]**

Solo falta ensamblar la tubería usando LangChain y el modelo de chat (gpt-4o-mini).

```python
from langchain_openai import ChatOpenAI
from langchain.schema.runnable import RunnablePassthrough
from langchain.schema.output_parser import StrOutputParser

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)

# Configuramos la cadena RAG
def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

print("\n🤖 Pensando...")
respuesta = rag_chain.invoke(pregunta_usuario)
print("\nRESPUESTA DEL CHATBOT:")
print(respuesta)
```

**[VISUAL: La terminal imprime la respuesta correcta basada exactamente en el PDF inventado]**

Al ejecutar esto, el código toma la pregunta, saca los 3 párrafos relevantes de Chroma, los empuja dentro de las llaves `{context}` de nuestro Prompt, se lo envía al LLM y nos escupe la respuesta exacta. ¡Sin alucinaciones!

El código de este laboratorio está disponible para que lo descargues. Felicidades, acabas de construir un sistema RAG empresarial funcional.

Pero... ¿qué pasa si el usuario no hace una pregunta directa, sino que dice *"Quiero pedir mis vacaciones ahora mismo, tramita el permiso"*? 
RAG solo responde preguntas; no puede ejecutar acciones. No puede enviar un correo, ni abrir un ticket de Jira.

Para que nuestro código Python empiece a tomar decisiones y hacer clic en botones por el usuario, tenemos que pasar del RAG simple a la construcción de **Agentes Autónomos (Agents) y Herramientas (Tools)**. Y eso es exactamente de lo que trata el Capítulo 4.
