### [01:00-02:15] CONFIGURANDO EL ESQUEMA DE SEGURIDAD (PYDANTIC)
**[VISUAL: El instructor crea un archivo `auditor.py` y empieza a escribir código Pydantic]**

Lo primero es definir qué queremos que la IA nos responda. Queremos una lista de vulnerabilidades encontradas, su nivel de riesgo y la línea de código donde está el error.

```python
from pydantic import BaseModel, Field
from typing import List

class Vulnerabilidad(BaseModel):
    linea_aprox: int = Field(description="La línea de código donde parece estar el error")
    descripcion: str = Field(description="Qué hace que esto sea inseguro")
    severidad: str = Field(description="ALTA, MEDIA o BAJA")
    sugerencia_fix: str = Field(description="Cómo debería escribirse correctamente")

class ReporteAuditoria(BaseModel):
    es_seguro: bool = Field(description="True si el código es perfecto, False si hay vulnerabilidades")
    vulnerabilidades_encontradas: List[Vulnerabilidad]
```

Con este esquema, le decimos al modelo: *"No me escribas un saludo. Entrégame un objeto ReporteAuditoria"*. Esto es magia arquitectónica, porque ahora tu código Python puede procesar la respuesta como objetos reales, no como strings crudos.

### [02:15-03:45] CONECTANDO A LA API Y FORZANDO EL FORMATO
**[VISUAL: Escribiendo la función de OpenAI usando las herramientas modernas de Parseo (Structured Outputs)]**

Ahora vamos a hacer la llamada a la API usando el SDK moderno de OpenAI, inyectando nuestra clase de Pydantic directamente en la petición.

```python
from openai import OpenAI
import os

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def auditar_codigo(codigo_fuente: str) -> ReporteAuditoria:
    print("Enviando código al auditor IA...")
    
    prompt = f"Actúa como un Ingeniero de Seguridad Senior. Audita este código:\n\n{codigo_fuente}"
    
    response = client.beta.chat.completions.parse(
        model="gpt-4o-mini", # Rápido y barato
        messages=[
            {"role": "system", "content": "Eres un auditor estricto de código en Python. Solo encuentras vulnerabilidades reales de seguridad, no problemas de formato PEP8."},
            {"role": "user", "content": prompt}
        ],
        response_format=ReporteAuditoria
    )
    
    # La respuesta ya viene tipada y validada
    return response.choices[0].message.parsed
```

Fíjense en el método `.parse()` y el parámetro `response_format`. Esto garantiza al 100% que la respuesta será un JSON que encaja perfectamente en nuestro objeto Pydantic. Si el LLM intenta alucinar un formato distinto, la API lo corregirá automáticamente.

### [03:45-04:45] EJECUTANDO LA PRUEBA CON CÓDIGO MALICIOSO
**[VISUAL: Ejecutando el script en la terminal y mostrando el resultado]**

Vamos a probarlo. Le voy a pasar un fragmento de código que tiene una inyección SQL clásica (concatenación de strings en una query de base de datos en lugar de usar parámetros).

```python
codigo_peligroso = """
def obtener_usuario(db_cursor, username):
    query = "SELECT * FROM users WHERE username = '" + username + "'"
    db_cursor.execute(query)
    return db_cursor.fetchall()
"""

reporte = auditar_codigo(codigo_peligroso)

if not reporte.es_seguro:
    print(f"⚠️ ¡ALERTA! Se encontraron {len(reporte.vulnerabilidades_encontradas)} fallos:")
    for vuln in reporte.vulnerabilidades_encontradas:
        print(f"[{vuln.severidad}] Línea {vuln.linea_aprox}: {vuln.descripcion}")
        print(f"👉 Fix: {vuln.sugerencia_fix}")
```

Al ejecutarlo en la terminal, ¡boom! La IA detecta la inyección SQL, la clasifica como severidad ALTA y nos sugiere usar sentencias parametrizadas. Todo parseado limpiamente para que podamos detener el pipeline de integración continua si queremos.
