### [01:00-02:20] PASO 1: CREANDO LA HERRAMIENTA (TOOL)
**[VISUAL: El instructor escribe código Python definiendo una función decorada con `@tool`]**

Un agente no es nada sin herramientas. Vamos a crear la herramienta que el modelo usará para ejecutar SQL. 
Aquí es vital documentar la función perfectamente (usando `Docstrings` y Type Hints de Pydantic), porque esa documentación es la que el LLM leerá para aprender cómo usar la herramienta.

```python
import sqlite3
from langchain_core.tools import tool

@tool
def ejecutar_consulta_sql(query: str) -> str:
    """
    Ejecuta una consulta SQL SELECT en la base de datos 'ventas.sqlite' 
    y devuelve los resultados. NUNCA uses comandos DELETE o UPDATE.
    La tabla principal se llama 'ventas' y tiene las columnas: 
    id, producto, precio, fecha.
    """
    print(f"\n[🔧 TOOL EJECUTANDO] SQL Query: {query}")
    try:
        conn = sqlite3.connect("ventas.sqlite")
        cursor = conn.cursor()
        cursor.execute(query)
        resultados = cursor.fetchall()
        conn.close()
        return str(resultados)
    except Exception as e:
        return f"Error SQL: {str(e)}"
```

Fíjense en el Docstring. Le estamos diciendo al modelo explícitamente cómo se llama la tabla y qué columnas tiene, para que no tenga que "adivinar" el esquema de la base de datos y alucinar nombres de columnas. También pusimos una advertencia de seguridad contra borrados.

### [02:20-03:40] PASO 2: INICIALIZANDO EL AGENTE Y BINDING DE HERRAMIENTAS
**[VISUAL: El instructor carga el LLM y hace el "bind" de la herramienta]**

Ahora necesitamos cargar el modelo. Para agentes, necesitas un modelo muy inteligente, ya que un modelo pequeño se perderá en la lógica. Usaremos `gpt-4o`.

```python
from langchain_openai import ChatOpenAI
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.prompts import ChatPromptTemplate

# 1. Cargamos el modelo
llm = ChatOpenAI(model="gpt-4o", temperature=0)

# 2. Le "atamos" (bind) nuestra herramienta al modelo
tools = [ejecutar_consulta_sql]

# 3. Creamos el Prompt del Agente
prompt = ChatPromptTemplate.from_messages([
    ("system", "Eres un analista de datos. Usa las herramientas provistas para buscar información en la base de datos y responderle al usuario en tono corporativo."),
    ("user", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
```

La magia ocurre en `{agent_scratchpad}`. Esa es la memoria a corto plazo del agente, donde guardará sus ciclos de Pensamiento, Acción y Observación.

---

### [03:40-04:40] PASO 3: EL BUCLE DE EJECUCIÓN (AGENT EXECUTOR)
**[VISUAL: Se crea el AgentExecutor y se lanza la prueba]**

LangChain tiene una clase llamada `AgentExecutor`. Es el motor que controla el ciclo "ReAct" que vimos en la teoría, asegurando que el agente siga intentando hasta resolver el problema.

```python
# 4. Construimos el Agente
agente = create_tool_calling_agent(llm, tools, prompt)

# 5. Creamos el Ejecutor
ejecutor = AgentExecutor(agent=agente, tools=tools, verbose=True)

# PRUEBA FINAL
pregunta = "¿Cuánto fue el total de ventas del producto 'Laptop Pro'?"
print(f"Pregunta del usuario: {pregunta}")

respuesta = ejecutor.invoke({"input": pregunta})
print("\n[🎯 RESPUESTA FINAL]:")
print(respuesta["output"])
```

### [04:40-06:00] VIENDO LA MAGIA EN ACCIÓN
**[VISUAL: Se ejecuta el script en la terminal. El texto aparece en color verde indicando los pensamientos y acciones del modelo en tiempo real]**

Al ejecutar el código, mira lo que pasa en la terminal:
1. El Agente recibe la pregunta. Piensa: *"Necesito sumar el precio de las ventas del producto 'Laptop Pro'."*
2. Decide llamar a la herramienta `ejecutar_consulta_sql`.
3. El Agente escribe su propio código SQL: `SELECT SUM(precio) FROM ventas WHERE producto = 'Laptop Pro';`
4. Nuestra función en Python la ejecuta y le devuelve la observación: `[(45000,)]`.
5. El Agente procesa el número y escribe la respuesta final humana: *"El total de ventas del producto 'Laptop Pro' fue de 45,000 dólares."*

El agente navegó la ambigüedad, usó una herramienta externa y resolvió el problema sin que nosotros programáramos ni un solo `if` ni una sola sentencia SQL explícita. Eso es el futuro de la arquitectura de software.
