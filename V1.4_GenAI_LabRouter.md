### [00:08-01:00] INTRODUCCIÓN AL LABORATORIO
**[VISUAL: Pantalla completa del IDE (VS Code o similar). El instructor empieza con un archivo vacío `main.py`]**

Bienvenido al laboratorio. Como vimos en el video anterior, enviar *todas* las peticiones de los usuarios al modelo más caro e inteligente del mercado (como GPT-4) es un error arquitectónico que va a sangrar el presupuesto de tu empresa.

Si el usuario solo dice "Hola", no necesitas una IA con un IQ altísimo para responder "Hola, ¿en qué te ayudo?". Pero si el usuario dice "Analiza esta base de datos financiera", ahí sí necesitas la artillería pesada.

Hoy vamos a programar en Python, utilizando **FastAPI**, un "Router de Modelos". Este router evaluará la longitud o complejidad del prompt del usuario y decidirá automáticamente si redirige la petición a un modelo "Barato y Rápido" o a un modelo "Complejo y Costoso".

Empecemos.

### [01:00-02:30] CONFIGURANDO EL BOILERPLATE Y FASTAPI
**[VISUAL: Escribiendo código en pantalla, importando librerías]**

Primero, importamos lo básico. Necesitamos FastAPI para exponer el servidor web, y Pydantic para validar los datos que nos envía el cliente.

*(El código se muestra en pantalla y el narrador explica sobre él)*

```python
from fastapi import FastAPI
from pydantic import BaseModel
import time

app = FastAPI(title="GenAI Model Router")

# Estructura de la petición del cliente
class UserRequest(BaseModel):
    prompt: str
```

Aquí tenemos nuestro servidor básico. El cliente nos enviará un JSON con la propiedad `prompt`. 

Ahora, vamos a simular nuestras dos APIs de Inteligencia Artificial. En la vida real aquí usarías la librería de OpenAI o Google, pero para mantener este código limpio y enfocado en la arquitectura, haremos funciones simuladas (Mock).

```python
def call_cheap_model(prompt: str) -> str:
    # Simula una llamada a GPT-3.5 o Claude Haiku
    time.sleep(0.5) # Baja latencia
    return f"[Modelo Barato] Entendido. Respondiendo a: {prompt}"

def call_expensive_model(prompt: str) -> str:
    # Simula una llamada a GPT-4 Turbo o Claude Opus
    time.sleep(2.5) # Alta latencia computacional
    return f"[Modelo Complejo] Análisis profundo ejecutado para: {prompt}"
```

### [02:30-03:45] CREANDO LA LÓGICA DEL ROUTER
**[VISUAL: Centrado en la nueva función de "Enrutamiento" que se está escribiendo]**

Aquí viene el cerebro de nuestra arquitectura. 

Vamos a crear una función que determine la complejidad. La forma más sencilla de empezar (el MVP) es por longitud. Si el prompt tiene más de 20 palabras, asumiremos que es una instrucción compleja o un bloque de texto que requiere análisis profundo.

```python
def route_model(prompt: str):
    word_count = len(prompt.split())
    
    if word_count > 20:
        print("Enrutando a modelo complejo (Alto Costo)...")
        return call_expensive_model(prompt)
    else:
        print("Enrutando a modelo barato (Bajo Costo)...")
        return call_cheap_model(prompt)
```

En un nivel más avanzado (producción), en lugar de contar palabras, podríamos usar un pequeño modelo de Machine Learning local para clasificar la *intención* del prompt (Intent Classification) en milisegundos, antes de decidir el ruteo.

### [03:45-04:45] EXPONIENDO EL ENDPOINT
**[VISUAL: Completando el código de FastAPI]**

Finalmente, creamos nuestro endpoint POST. Este es el puerto donde nuestra aplicación de Front-End o nuestra App Móvil se conectará.

```python
@app.post("/chat")
async def chat_endpoint(request: UserRequest):
    
    # El router hace la magia de forma transparente para el cliente
    respuesta_ia = route_model(request.prompt)
    
    return {
        "status": "success",
        "response": respuesta_ia
    }
```

¡Y eso es todo! 

Si ejecutas este servidor con Uvicorn y le envías un simple "Hola, buen día", tu backend gastará fracciones de centavo usando el modelo barato y responderá en 500 milisegundos.

Si le pegas un párrafo legal de 50 palabras pidiendo un resumen, el Router lo detectará y abrirá la válvula del modelo costoso, asegurando que la calidad de la respuesta sea la adecuada.
