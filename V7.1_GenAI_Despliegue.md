Hola, arquitectos. Bienvenidos al Capítulo 7, nuestro módulo final. Es momento de llevar todo lo que hemos construido en nuestro entorno local hacia un entorno de producción real, seguro y escalable.

**[VISUAL: Slide 1 - Contenedores y Docker]**
No podemos ejecutar una API de inteligencia artificial directamente en un servidor virtual usando 'python run'. Si el tráfico aumenta, el servidor colapsará. La solución estándar en la industria es la contenedorización con Docker. Docker empaca nuestro código de Python, sus librerías como LangChain o FastAPI, y el sistema operativo base, en un paquete inmutable que correrá exactamente igual en tu computadora que en los servidores de Amazon AWS o Microsoft Azure.

**[VISUAL: Slide 2 - Seguridad en Producción]**
Cuando tu aplicación sale a internet, los ataques son inevitables. El más común en GenAI es el 'Prompt Injection', donde un usuario malicioso intenta engañar a tu modelo para que ignore sus instrucciones y ejecute un ataque. Además, jamás debes incluir tus API Keys en el código fuente ni en el Dockerfile. Siempre debemos usar inyección de secretos mediante variables de entorno o sistemas como AWS Secrets Manager.
