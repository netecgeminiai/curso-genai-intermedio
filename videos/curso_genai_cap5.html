1. Fine-Tuning vs RAG

RAG: Ideal para inyectar datos privados o nuevos conocimientos.

Fine-Tuning: Ideal para enseñar un "comportamiento" o "tono" específico al modelo.

Ejemplo: Si quieres respuestas estructuradas en JSON perfecto = Fine-Tuning.

Ejemplo: Si quieres el saldo de un cliente de hoy = RAG.

[+] MATRIZ DE DECISIÓN

■ Comportamiento / Tono → Fine-Tuning
■ Conocimiento Actualizado → RAG
■ Costo Computacional → FT es alto, RAG es bajo
■ Mejor Práctica → FT para formato + RAG para datos

2. Datasets JSONL

Para entrenar requerimos miles de ejemplos.

Formato JSONL: JSON Lines. Cada línea es un JSON independiente.

Debe incluir System, User y Assistant para imitar la conversación completa.

{"messages": [{"role": "system", "content": "Habla como pirata"}, {"role": "user", "content": "Hola"}, {"role": "assistant", "content": "¡Ahoy marinero!"}]}

{"messages": [{"role": "system", "content": "Habla como pirata"}, {"role": "user", "content": "Adiós"}, {"role": "assistant", "content": "¡Que los vientos te sean favorables!"}]}
