1. Orquestación: LangChain vs LlamaIndex

El problema: Escribir RAG desde cero es frágil. Necesitamos un Framework de Orquestación.

LangChain: Propósito general, integración masiva (500+ herramientas), ideal para Agentes y Memoria.

LlamaIndex: El rey del RAG. Ingesta brutal de datos y estructuración de índices avanzada.

[+] LA DECISIÓN DEL ARQUITECTO

■ ¿Chatbot creativo o Agente que toma acciones?
  > Usa LangChain.

■ ¿Buscador semántico sobre 1,000 PDFs densos?
  > Usa LlamaIndex.

■ Nivel Enterprise:
  > LlamaIndex para recuperar datos (R) + LangChain como Agente orquestador.

2. Agentes y Tools

De respuesta a acción: RAG solo lee. Los Agentes toman decisiones y ejecutan código.

Bucle ReAct: Thought -> Action -> Observation -> (Loop).

Tools (Function Calling): Funciones de Python decoradas y enviadas al LLM en el System Prompt.

Seguridad: Human-In-The-Loop y Sandboxing.
