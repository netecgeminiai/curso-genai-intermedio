1. Orquestación: LangChain vs LlamaIndex

El problema: Escribir RAG desde cero es frágil. Necesitamos un Framework de Orquestación.

LangChain: Propósito general, integración masiva (500+ herramientas), ideal para Agentes y Memoria.

LlamaIndex: El rey del RAG. Ingesta brutal de datos y estructuración de índices avanzada.

[+] LA DECISIÓN DEL ARQUITECTO

■ ¿Chatbot creativo o Agente que toma acciones?
  > Usa LangChain.

■ ¿Buscador semántico sobre 1,000 PDFs densos?
  > Usa LlamaIndex.

■ Nivel Enterprise:
  > LlamaIndex para recuperar datos (R) + LangChain como Agente orquestador.

2. Agentes y Tools

De respuesta a acción: RAG solo lee. Los Agentes toman decisiones y ejecutan código.

Bucle ReAct: Thought -> Action -> Observation -> (Loop).

Tools (Function Calling): Funciones de Python decoradas y enviadas al LLM en el System Prompt.

Seguridad: Human-In-The-Loop y Sandboxing.

[ User: "Cancela mi suscripción" ]
⬇
[ Agente: Piensa ] "Debo usar la Tool StripeAPI"
⬇
[ Agente: Action ] Llama a `cancelar_sub(id)`
⬇
[ Python: Ejecuta API Stripe ] Devuelve "OK"
⬇
[ Agente: Respuesta final ] "Suscripción cancelada."

3. Lab: Agente Autónomo SQL

Construyendo un Agente que habla directamente con nuestra Base de Datos `sqlite`.

1. Definir la Tool con Docstrings estrictos.

2. Hacer bind de la Tool al LLM.

3. Inicializar el `AgentExecutor`.

El Agente escribirá sus propias queries SQL.

import sqlite3 from langchain_core.tools import tool @tool def ejecutar_sql(query: str) -> str: """ Ejecuta una consulta SQL en la BD. La tabla 'ventas' tiene: id, producto, precio. NUNCA uses comandos DELETE o UPDATE. """ conn = sqlite3.connect("ventas.db") cursor = conn.cursor() cursor.execute(query) return str(cursor.fetchall())
